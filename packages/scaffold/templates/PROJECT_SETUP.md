# Project Setup and Development Workflow

## Welcome

This guide helps you set up a production-ready, developer-friendly
codebase. Answer the eight questions below, and a personalized
PROJECT_CONFIG.md will be generated to guide your workflow.

## Questionnaire

### Question 1: What is your project?

Describe briefly: project name, purpose, core idea.

### Question 2: What's your skill level?

- Beginner
- Intermediate
- Advanced

### Question 3: What kind of project is this?

- Backend Service
- Standalone Application
- Library / SDK
- CLI Tool
- Data Processing
- Other

### Question 4: What kind of software is this — where will it run?

- Desktop — Windows
- Desktop — macOS
- Desktop — Linux
- Desktop — cross-platform (Windows + macOS + Linux)
- Android app
- iOS app
- Web app (browser)
- Server / cloud
- Other / not sure

### Question 5: What's your primary development language?

- Python
- JavaScript / Node.js
- Rust
- Go
- Java
- Other

### Question 6: Do you have a containerization or virtualization environment available?

- Docker / container runtime
- Hypervisor (Proxmox, KVM, ESXi, Hyper-V, VirtualBox, etc.)
- Neither (local development only)

### Question 7: What's your primary development OS?

- Windows
- Linux
- macOS
- Other

### Question 8: Do you have any specific project philosophy, constraints, or custom logic for this project?

Describe briefly (e.g., "anti-e-waste hardware reuse", "minimal
dependencies", "GPU-accelerated", "real-time constraints").

## Next Steps

Once you've answered all eight questions, generate your
PROJECT_CONFIG.md by running the setup tool or MCP. This will create a
personalized configuration file specific to your project.

Your PROJECT_CONFIG.md will drive:

- Project scaffolding
- Dependency management
- Development environment setup
- Testing strategy
- Staging and production workflows
- Monitoring and observability
- Rollback procedures
- Learning loop documentation
